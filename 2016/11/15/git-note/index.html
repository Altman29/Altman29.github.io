Git 学习笔记

发表于 2016-11-15   |   分类于 Android   |     |   阅读次数

作为强大的版本管理软件，已经得到广泛的应用，网上也很多关于Git操作的讲解。也是因为这点，所以个人开篇就以此而立，本人也是经常使用，但着实研究的不深，Tell you the truth，经常是需要用到才Google一下。不知道有没有小伙伴和我一样式的。偶然发现，用了这么久，居然几句代码都没记住，捂脸。其实很多时候针对一些不常用的技术或者工具什么的，都抱着随时用随时查，它不就在那里么，还需要记么。时间长了，会有惰性，导致木有办法升职加薪，迎娶大美女，走向人生巅峰，细思恐极啊。也是最近一些事改变了我的想法，还专门用了1个周末的时间弄好了这个blog。In my house,哈哈，以后这就是本汪的小屋，记录生活钻研技术。想想还有点小激动呢。

我擦，跑偏了，不好意思。这就开始搞Git。

Git若是从基础说，那一篇blog也是说不完的。而且能力有限，就不写教课书了，我把我工作中常用的记录下来，以供日后使用。（别担心，若是你看不过瘾，文末有惊喜呢。）

github给予了我们不限量的空间来使用，今天我也来简单地总结一下在git远程使用时经常会用到的一些命令与技巧。

1.初次连接本地与远程主机

git init 
git add .
git commit -m "first commit"
git remote add origin git@github.com.. //将本地仓库与github连接，并创建主机origin  
git push origin master:maseter //将本地master推送到远程origin/master

2.git remote项目命令

git要求每一个远程主机都要有一个独立的主机名，而gitremote相关命令，就是用来管理主机名的。

git remote 用来查看当前项目的所有主机名
git remote -v 则可以显示主机名及其网址
git remote show [主机名] 用来查看某个主机的详细信息
git remote add [主机名][github网址] 用来添加一个远程主机
git remote rm [主机名] 则会删除相应的主机

3.git push相关命令

git push [远程主机名][本地分支名]:[远程分支名]//用于将本地提交内容推送到远程主机上
git push [远程主机名] //将当前分支推送到远程主机的对应分支

4.git pull相关命令

由于git fetch在取回服务器上的程序更新时，还需要手动进行合并，对于一些小型的程序而言，git pull命令则更为实用。

git pull [远程主机名][远程分支名]:[本地分支名]//取回远程某个分支更新，并与本地指定分支合并
git pull [远程主机名][远程分支名] 取回远程分支的更新，并与本地当前分支合并

5.关于忽略同步

在使用git的时候，我们往往希望在同步过程中忽略一些涉及私人信息的文件，如使用Mac时的.DS_Store文件。此时也很简单，只需要添加名为.gitignore的文件并在其中添加相应的忽略文件即可。
